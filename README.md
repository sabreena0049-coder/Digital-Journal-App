# 📖 StoryLeaf – Digital Journal Application

## Overview

StoryLeaf is a user-friendly digital journal application developed using Python and Streamlit. The application provides a simple and secure platform for users to record
their daily thoughts, experiences, memories, and reflections in a digital format. Instead of maintaining traditional paper diaries, users can conveniently create, store,
search, and manage journal entries through an interactive web interface.

The primary objective of StoryLeaf is to help users maintain a personal journal that is organized, accessible, and easy to manage. The application allows users to write
entries, track their moods, search through previous notes, view journal statistics, and delete entries when necessary. All data is stored locally using JSON files,
eliminating the need for complex database systems while ensuring simplicity and portability.


StoryLeaf demonstrates the practical implementation of file handling, data management, user interaction, and visualization techniques using Python and Streamlit. The
project serves as an excellent example of how modern web applications can be built with minimal dependencies and a clean architecture.


## Features

### ✍️ Create Journal Entries

Users can create new journal entries by providing a title, selecting their current mood, and writing detailed content. Each entry is automatically timestamped with the
current date and time, making it easy to track events chronologically.

### 😊 Mood Tracking

The application includes mood selection options such as Happy, Calm, Neutral, Sad, Angry, and Tired. This feature helps users reflect on their emotional state and monitor
mood patterns over time.

### 📚 View Saved Entries

All journal entries can be viewed in an organized and readable format. Entries are displayed with their title, mood, timestamp, and content, allowing users to revisit past
memories and experiences whenever needed.

### 🔍 Search Functionality

StoryLeaf includes a search feature that enables users to quickly find specific journal entries using keywords. The search operation scans both titles and content, making
information retrieval efficient and convenient.

### 🗑️ Delete Entries

Users can remove unwanted or outdated journal entries through the delete functionality. This ensures better organization and control over stored data.

### 📊 Statistics Dashboard

The application provides useful insights about journal activity, including:

* Total number of entries
  
* Mood distribution analysis
  
* Most recent journal entry information

These statistics help users understand their journaling habits and emotional trends.

## Project Architecture

StoryLeaf follows a simple single-file architecture. Unlike traditional web applications that require separate frontend and backend components, this project combines all
functionality within one Streamlit application. Data is stored locally in a JSON file, which acts as a lightweight storage system.

The application workflow consists of:

1. User interaction through the Streamlit interface.
   
3. Data processing using Python functions.
   
5. Storage and retrieval of journal entries using JSON files.
   
7. Visualization of statistics through Streamlit components.

This architecture makes the project easy to understand, deploy, and maintain.
